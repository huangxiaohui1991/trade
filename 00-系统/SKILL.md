# A股交易助手 Skill

## 触发关键词
选股打分、计算仓位、止损计算、止盈计算、加仓判断、风控检查、今日检查、周复盘、交易记录、持仓查询、核心池、观察池、交易系统、买入检查、卖出检查、明日计划、收盘记录、盘前检查、午休检查

## 功能概述
基于交易系统手册 v1.3 的规则引擎，提供以下能力：
1. **仓位计算** — 根据总资金和持仓计算买入金额、止损价、止盈价
2. **选股打分** — 四维框架打分，自动判断是否通过
3. **加仓判断** — 检查加仓条件、次数限制、仓位限制
4. **风控检查** — 冷却机制、大盘状态、仓位超限检查
5. **每日检查单** — 生成盘前/午休/收盘待办
6. **每周复盘** — 统计胜率、盈亏比，生成复盘报告
7. **交易记录** — 记录买卖操作到 Obsidian MD 文件

## 计算脚本
- `scripts/calculator.py` — 仓位、止损、止盈、加仓计算
- `scripts/scorer.py` — 选股打分引擎
- `scripts/parser.py` — MD 文件解析（YAML Front Matter + 表格）
- `scripts/akshare_data.py` — AKShare 数据获取（行情、基本面、资金流向、自动打分）

## 使用方式

### 1. 选股打分
用户说"帮我给 XXX 打分"或"选股打分"时：
1. 运行 `python scripts/akshare_data.py score <股票代码>` 自动获取数据并打分
2. 自动评分项（技术面、成交量、资金面）直接出结果
3. 需人工确认项（行业逻辑、催化、ROE、营收）提示用户补充
4. 输出完整打分表和建议
5. 如果通过，询问是否加入观察池/核心池

批量打分：`python scripts/akshare_data.py batch_score 600001,000002,601127`

### 2. 计算仓位/买入检查
用户说"计算仓位"或"我想买 XXX"时：
1. 读取 `data/01-持仓/portfolio.md` 获取当前持仓
2. 运行 `python scripts/calculator.py` 计算
3. 输出：建议买入金额、止损价、三批止盈价、加仓触发价
4. 检查是否超过仓位限制、本周买入次数

### 3. 加仓判断
用户说"XXX 可以加仓吗"时：
1. 读取持仓数据，获取首次买入价和当前持仓
2. 运行计算脚本检查加仓条件
3. 输出：是否满足加仓条件、建议加仓金额、加仓后新止损价
4. 检查本周买入次数和单只加仓次数

### 4. 风控检查
用户说"风控检查"时：
1. 读取持仓和本周交易记录
2. 检查所有风控规则
3. 输出：是否触发冷却、仓位状态、大盘状态、买入次数余量

### 5. 每日检查单
用户说"今日检查"或"盘前检查"时：
- 盘前：读取今日日志中的明日计划（昨晚生成），补充实时大盘数据，输出盘前摘要
- 午休：检查持仓状态，提醒加仓机会
- 收盘：更新日志，生成明日计划写入明天的日志文件

用户说"收盘记录"时：
1. 运行 `akshare_data.py market_status` 获取大盘收盘状态
2. 运行 `akshare_data.py technical <持仓代码>` 更新持仓最新价
3. 重算所有止损/止盈价（calculator.py）
4. 检查时间止损（持有天数 + 振幅）
5. 检查核心池标的是否跌破 20 日线
6. 更新今日日志的收盘部分
7. **生成明日计划**（写入明天的日志文件）：
   - 大盘状态 + 是否可操作
   - 风控状态（买入次数余量、仓位、冷却）
   - 条件单清单（每只持仓的止损价、止盈价、需要挂哪些单）
   - 核心池买入条件检查（距离买入条件还差多少）
   - 持仓关注事项（时间止损倒计时、加仓条件、止盈接近）
   - 明日重点（1-3 条关键行动）

用户说"明日计划"时：
- 等同于收盘记录的第 7 步，单独生成明日计划部分

### 6. 每周复盘
用户说"周复盘"时：
1. 读取本周所有日志文件
2. 运行计算脚本统计胜率、盈亏比
3. 生成周复盘 MD 文件
4. 提醒更新核心池

### 7. 记录交易
用户说"记录买入/卖出"时：
1. 收集交易信息（股票、价格、数量、类型）
2. 更新 portfolio.md
3. 追加到当日日志
4. 更新 JSON 缓存

## 交易系统规则 v1.4（回测优化版）

> 以下是 Skill 执行所有计算和检查的规则依据。v1.4 在 v1.3 基础上通过历史回测优化了参数。
> 回测验证：杰瑞股份 +15.1%/盈亏比3.19，大金重工 +8.1%/盈亏比2.42，顾家家居 +1.1%/盈亏比2.40

### 核心原则
1. 资金安全第一：任何一笔交易的最大亏损不超过总资金的 2%
2. 系统大于预测：不猜顶底，只按信号买卖
3. 情绪隔离：生气、恐慌、极度兴奋时禁止交易
4. 持续迭代：每周复盘，每月优化系统参数
5. 自动化优先：能用条件单的，绝不用手动盯盘

### 选股规则

#### 四维框架（至少满足 3 个维度）
| 维度 | 权重 | 核心指标 | 通过标准 |
|------|------|----------|----------|
| 行业逻辑 | 30% | 政策支持、机构共识、景气度 | 属于当前市场主线 |
| 基本面 | 30% | ROE、营收增长、现金流 | ROE≥8%（两年），营收连续两季正增长，经营现金流为正 |
| 技术面 | 25% | 均线、成交量 | 股价站上20日均线，60日均线向上或走平 |
| 主力资金 | 15% | 北向资金、主力净流入 | 近5日北向净流入，无连续3日大额主力流出 |

#### 打分表（10 项 + 1 加分项）
1. 属于当前主线板块（1分）
2. 有政策或产业催化（1分）
3. ROE ≥ 8%（连续两年）（1分）
4. 营收连续两季度增长（1分）
5. 经营现金流为正（1分）
6. **股价站上 20 日均线（1分，一票否决项）**
7. 60 日均线向上或走平（1分）
8. 成交量条件：当日>20日均量(1分) / 近3日曾放量(0.5分) / 否则(0分)，要求≥0.5
9. 北向资金近 5 日净流入（1分）
10. 无连续主力大额卖出（1分）
11. 加分项：突破当日成交量 > 5日均量的1.5倍（+0.5分）

**通过标准**：≥ 7 分，且第 6 项（站上20日线）必须满足。

#### 备选池管理

**定期筛选（每周日）：**
- 观察池：用 i问财 分层筛选 10-15 只，batch_score 打分后保留 5 只
- 核心池：从观察池选出 3 只，作为下周重点标的

**事件驱动更新（随时）：**
- 重大合同/订单公告 → 检查是否在池中，不在则临时打分评估是否加入
- 业绩暴雷/重大利空 → 立即从核心池/观察池剔除
- 板块政策催化 → 快速扫描相关板块龙头，评估是否加入观察池
- 核心池标的跌破 20 日线 → 剔除，从观察池补位
- 大盘状态变化（站上/跌破均线） → 重新评估操作策略

**上班族执行原则：** 盘前只看核心池+观察池相关新闻标题（5分钟），重大事件午休评估，其他消息等周日统一处理。不因消息焦虑而临时决策。

**剔除规则：** 跌破 20 日线超过 3 天、业绩暴雷、主线切换、连续 3 日主力大额流出

### 资金管理

| 项目 | 规则 |
|------|------|
| 账户总资金 | 80 万（可配置） |
| 股票仓位上限 | 60%（48 万） |
| 单只最大仓位 | 20%（16 万） |
| 单笔买入金额 | 2-3 万（首次）/ 2-3 万（加仓） |
| 现金配置 | ≥ 40% 配置于国债逆回购/货币基金 |

### 买入规则

#### 首次买入条件（全部满足）
1. 标的来自核心池，打分 ≥ 7 分
2. 大盘条件：
   - 标准：上证或创业板站上 20 日均线
   - 例外：大盘跌破20日线但仍在60日线上方，且个股满足全部三条件（站上所有短期均线、强于板块5日涨幅超5个百分点、连续3日放量），单笔金额减半至 1-1.5 万
3. 个股距离最近买入信号不超过 3 个交易日
4. 当日无重大利空
5. `[v1.4]` **2日确认**：连续 2 天收盘在 20 日线上方才买入（避免假突破）
6. `[v1.4]` **追涨限制**（全部满足）：
   - 非趋势股：买入价不超过 20 日线的 8%
   - 趋势股（均线发散度 > 5%）：买入价不超过 20 日线的 15%（放宽，允许追趋势）
   - 当日涨幅 ≤ 5%（涨停板不追）
   - 近 3 日累计涨幅 ≤ 10%（连续拉升不追）
7. `[v1.4]` **连续止损冷却**：同一只股票连续 2 次止损后，15 个交易日内不再买入（防假突破兜底）

#### 加仓规则
| 时机 | 条件 | 金额 |
|------|------|------|
| 第一次加仓 | 从首次买入价上涨 ≥7%，且站上20日线 | 2-3万（≤首次买入额） |
| 第二次加仓 | 从首次买入价上涨 ≥12%，且60日线向上 | 2-3万 |
| 更多加仓 | 每再涨5%可加一次 | 2-3万 |

**加仓限制：**
- 加仓计入"每周最多买入 2 次"限额
- 同一只股票每周最多加仓 1 次
- 加仓后止损价按平均成本重新计算
- `[v1.3]` **加仓前风险检查**：加仓后如果触发止损，总亏损必须 < 总资金的 2%，否则不加仓
- `[v1.3]` **加仓优先级**：多只股票同时满足加仓条件时，优先加仓趋势强度最高的（涨幅/回撤比最大）

### 卖出规则

#### 硬止损（v1.4 三重保护）
```
止损价 = MAX(
  平均成本价 × 0.96,          # 4% 成本止损（v1.4 从5%收紧）
  当前20日线价格 × 0.97       # 3% 均线止损
)
```
- **绝对止损线**：平均成本价 × 0.93（-7%），无条件止损，不受其他条件影响
- 加仓后重新计算平均成本，止损价随之调整

#### 止盈（v1.4 分批 + 移动止盈）

**涨停保护：** 当日涨幅达到涨停（主板≥9.5%，创业板/科创板≥19.5%）时，不触发任何止盈卖出。

**趋势股强势不卖：** 均线发散度（MA5-MA60)/MA60 > 5% 时判定为趋势股，延迟第一批止盈，让利润跑。但涨幅超过 +30% 时强制卖出 1/4（安全阀）。

| 批次 | 非趋势股 | 趋势股（发散度>5%） |
|------|---------|-------------------|
| 第一批 | 从首次买入价上涨 15% 卖 1/4 | 延迟止盈，+30% 强制卖 1/4 |
| 第二批 | 剩余仓位从最高点回撤 5% 卖 1/3 | 同左（强势不保护第二批） |
| 第三批 | 跌破20日线 或 从最高点回撤 8% 清仓 | 同左 |

#### 时间止损
- 买入后 5 个交易日内**振幅 < 4%**（v1.4 从3%放宽），主动卖出

#### 大盘止损
- `[v1.4]` 大盘跌破 20 日线 → **减仓到 50%**（卖出每只持仓的一半，每只只减一次）
- 大盘**连续 3 日收盘**在 60 日线下方 → 清仓所有股票（避免假跌破）
- **熊市空仓规则**：一个月内因大盘止损被迫减仓 2 次以上 → 进入"空仓观望模式"，等大盘站上 60 日线再回来

### 交易频率与持仓限制
| 项目 | 规则 |
|------|------|
| 每周最多主动买入次数 | 2 次（首次买入 + 加仓都计入） |
| 同时持仓股票数量 | ≤ 3 只 |
| 单只每周最多加仓次数 | 1 次 |
| 卖出不计入次数 | 止盈、止损等卖出不限 |

### 情绪控制与冷却机制

#### 强制冷却条件（触发任一 → 停止交易 3 天）
- 单日账户总资产回撤 > 3%
- 连续两笔交易触发止损
- 出现情绪化冲动
- 未按系统规则交易

### 紧急情况处理
| 情况 | 操作 |
|------|------|
| 账户从高点回撤 8% | 全部清仓，100% 理财，休息 1 个月 |
| 连续 3 笔止损 | 暂停 1 周，只做复盘 |
| 踏空焦虑 | 买入 1 天期逆回购，锁仓 3 天 |

### 遗留仓位（赛力斯专项）
- 16 万独立管理，跌 8% 清仓，涨 10% 卖 1/3，60 交易日清仓
- 禁止用新系统资金补仓
