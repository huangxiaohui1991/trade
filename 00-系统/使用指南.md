# A股交易系统 · Hermes 原生架构使用指南

> 版本：v2.0（Hermes-native）
> 更新日期：2026-04-09
> 分支：`feature/agent-native`

---

## 一、系统概述

本系统是基于 **Hermes 单 Agent 架构**的 A股自动化交易系统：

- **单 Agent 驱动**：Hermes 负责 cron 调度、Discord 推送、评分计算、Obsidian 读写
- **数据源**：东方财富（AKShare）优先，新浪作为 fallback
- **四维评分**：技术面(2) + 基本面(3) + 资金(2) + 舆情(3) = 10 分
- **风控核心**：4% 止损 / 7% 绝对止损 / 60% 仓位上限 / 每周最多 2 笔
- **唯一事实来源**：Obsidian Vault（持仓/日志/评分/配置均读写此处）

## 三、配置文件说明

### 3.1 `config/strategy.yaml` — 策略参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `capital` | ¥450,286 | 总资金 |
| `scoring.weights.technical` | 2 | 技术面权重（满分 2） |
| `scoring.weights.fundamental` | 3 | 基本面权重（满分 3） |
| `scoring.weights.flow` | 2 | 资金流权重（满分 2） |
| `scoring.weights.sentiment` | 3 | 舆情权重（满分 3） |
| `scoring.thresholds.buy` | ≥7 分 | 可买入 |
| `scoring.thresholds.watch` | ≥5 分 | 可观察池 |
| `risk.stop_loss` | 4% | 动态止损（成本 × 0.96 或 MA20 × 0.97） |
| `risk.absolute_stop` | 7% | 绝对止损（无条件，成本 × 0.93） |
| `risk.take_profit.t1_pct` | +15% | 第一批止盈（涨 15% 卖 1/3） |
| `risk.take_profit.t1_drawdown` | -5% | 第二批回撤（从高点回 5% 卖 1/3） |
| `risk.take_profit.t2_drawdown` | -8% | 第三批清仓回撤 |
| `risk.time_stop_days` | 15 | 时间止损（持有 15 日且涨幅 < 2% 止损） |
| `risk.position.total_max` | 60% | 总仓位上限 |
| `risk.position.single_max` | 20% | 单只仓位上限 |
| `risk.position.weekly_max` | 2 | 每周最多买入次数（含加仓） |
| `market_timer.green_days` | 3 日 | 连续 3 日站上 MA20 → GREEN |
| `market_timer.red_days` | 5 日 | 连续 5 日跌破 MA20 → RED |
| `market_timer.clear_days_ma60` | 15 日 | MA60 下方 15 日 → CLEAR |

> **修改规则**：修改 `strategy.yaml` 后立即生效，无需重启任何服务。

## 八、核心池 / 观察池管理

### 8.1 入池标准

- [ ] 四维评分 ≥ 7 分
- [ ] 价格 > MA20（站上20日线）
- [ ] 基本面 ≥ 1 分（ROE ≥ 8% 或营收增长 ≥ 10%）
- [ ] 近 5 日无连续主力大额流出
- [ ] 非高位放量整理（连续拉升 > 30% 谨慎追入）

### 8.2 操作

编辑 `config/stocks.yaml` 或在 Discord 说：
```
大金重工加入核心池
大金重工移出核心池
```

## 九、四维评分详解

满分 10 分，各维度独立计算后加权汇总：

| 维度 | 权重 | 满分 | 核心指标 |
|------|------|------|---------|
| 技术面 | 2 | 2.0 | MA20/MA60/成交量/动量/均线排列 |
| 基本面 | 3 | 3.0 | ROE/营收增长/现金流 |
| 资金流 | 2 | 2.0 | 主力净流入/北向资金 |
| 舆情 | 3 | 3.0 | TrendRadar（待接入，默认 1.5 分） |

### 一票否决（任一触发直接不买）

- `below_ma20`：价格 < MA20
- `limit_up_today`：今日涨停（涨幅 ≥ 9.9%）
- `consecutive_outflow`：连续 3 日主力大额流出
- `red_market`：大盘信号 = RED

## 十、风控规则

### 止损
```
动态止损 = MAX(成本 × 0.96, MA20 × 0.97)
绝对止损 = 成本 × 0.93（无条件，不受其他条件影响）
```

### 止盈（三批）
| 批次 | 触发条件 | 卖出比例 |
|------|---------|---------|
| 第一批 | 从买入价上涨 +15% | 1/3 |
| 第二批 | 从最高点回撤 -5% | 1/3 |
| 第三批 | 跌破 MA20 或从最高点回撤 -8% | 全部 |

> **涨停保护**：主板涨幅 ≥ 9.5%、创业板/科创板 ≥ 19.5% 时，不触发任何止盈卖出。

### 仓位限制
- 总仓位上限：60%
- 单只仓位上限：20%
- 每周最多买入：2 次（加仓也计入）
- 同时持仓：≤ 3 只

## 十二、常见问题

**Q：Cron 没有触发？**
A：确认 cron 已加载：`crontab -l`。确认 Hermes 守护进程在运行。

**Q：Discord 没有收到推送？**
A：确认 `DISCORD_WEBHOOK_URL` 环境变量已由 launchd 注入。检查 `notification.yaml` 中对应任务 `enabled: true`。

**Q：评分显示数据错误？**
A：非交易时间网络波动导致。技术指标有多源 fallback（东财→新浪→历史），若全失败返回 `stale: true`，表示数据过期。

**Q：如何修改止损比例？**
A：编辑 `config/strategy.yaml` 的 `risk.stop_loss` 和 `risk.absolute_stop`，立即生效。

**Q：想临时禁止某只股票买入？**
A：在 `config/stocks.yaml` 的 `blacklist.temporary` 中加入代码：
```yaml
blacklist:
  temporary: [002487]  # 大金重工临时禁止买入
```

**Q：TrendRadar 舆情什么时候能用？**
A：待接入。目前舆情维度默认 1.5 分（中性）。接入后会自动更新评分。
