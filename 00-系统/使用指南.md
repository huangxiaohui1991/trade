# A股交易系统 · 使用指南

> 版本：v2.2（共享 CLI / workflow 版）
> 更新日期：2026-04-09
> 本文档是系统唯一操作手册，涵盖交易规则 + 系统使用 + 配置说明。

---

## 一、系统概述

- **共享编排层**：Hermes-Agent 与 OpenClaw 共用 `bin/trade`
- **数据源优先级**：妙想 API（东方财富官方）→ AKShare → 新浪
- **四维评分**：技术面(2) + 基本面(3) + 资金(2) + 舆情(3) = 满分 10
- **唯一事实来源**：Obsidian Vault

## 二、选股规则

### 2.1 四维评分（满分 10 分）

| 维度 | 权重 | 满分 | 核心指标 |
|------|------|------|---------|
| 技术面 | 2 | 2.0 | 均线趋势 + 量比(按市值分档) + 5日动量 + MA5/MA20/MA60排列 |
| 基本面 | 3 | 3.0 | ROE + 营收增长率(MX补调) + 经营现金流(MX补调) |
| 资金流 | 2 | 2.0 | 换手率(3-10%最优) + 北向资金 |
| 舆情 | 3 | 3.0 | MX资讯搜索（研报数量 + 机构评级统计，只评前8名） |

### 2.2 一票否决（任一触发 → 总分归 0）

| 信号 | 条件 |
|------|------|
| `below_ma20` | 最新价 < 20日均线 |
| `limit_up_today` | 今日涨幅 ≥ 9.9% |
| `consecutive_outflow` | 连续 3 日主力大额流出（硬否决） |
| `consecutive_outflow_warn` | 连续 3 日流出但站稳 MA20 且成交额 > 5 亿 → 减 2 分不否决（疑似洗盘） |
| `red_market` | 大盘信号 = RED |
| `earnings_bomb` | 净利润同比下滑 > 30% |

### 2.3 分池规则

| 总分 | 去向 |
|------|------|
| ≥ 7 分 | 核心池（≤ 3 只） |
| 5-7 分 | 观察池（≤ 5 只） |
| < 5 分 | 淘汰 |

全部 < 7 分时核心池留空，不强行买入。选股完自动同步东方财富自选股（增删保持一致）。

> 详细评分逻辑见 `docs/选股评分逻辑.md`

## 三、买入规则

### 3.1 首次买入条件（全部满足）

1. 标的来自核心池，评分 ≥ 7 分
2. 大盘信号 GREEN（上证或创业板站上 MA20）
3. **2日确认**：连续 2 天收盘在 20 日线上方
4. 当日无重大利空
5. **追涨限制**：
   - 非趋势股：买入价不超过 MA20 的 8%
   - 趋势股（均线发散度 > 5%）：不超过 MA20 的 15%
   - 当日涨幅 ≤ 5%（涨停不追）
   - 近 3 日累计涨幅 ≤ 10%
6. **连续止损冷却**：同一只股票连续 2 次止损后，15 个交易日内不再买入

### 3.2 加仓规则

| 时机 | 条件 | 金额 |
|------|------|------|
| 第一次加仓 | 从首次买入价上涨 ≥7%，且站上 MA20 | 2-3万（≤首次买入额） |
| 第二次加仓 | 从首次买入价上涨 ≥12%，且 MA60 向上 | 2-3万 |
| 更多加仓 | 每再涨 5% 可加一次 | 2-3万 |

加仓限制：
- 计入"每周最多 2 次"限额
- 同一只股票每周最多加仓 1 次
- 加仓后止损价按平均成本重新计算
- 加仓后如果触发止损，总亏损必须 < 总资金的 2%

### 3.3 大盘例外

大盘跌破 MA20 但仍在 MA60 上方，且个股同时满足：站上所有短期均线、强于板块 5 日涨幅超 5 个百分点、连续 3 日放量 → 可买入，但单笔金额减半至 1-1.5 万。

## 四、卖出规则

### 4.1 止损（三重保护）

```
动态止损 = MAX(平均成本 × 0.96, 当前MA20 × 0.97)
绝对止损 = 平均成本 × 0.93（无条件，不受其他条件影响）
```

### 4.2 止盈（分批 + 移动止盈）

**涨停保护**：主板 ≥ 9.5%、创业板/科创板 ≥ 19.5% 时不触发止盈。

| 批次 | 非趋势股 | 趋势股（发散度>5%） |
|------|---------|-------------------|
| 第一批 | +15% 卖 1/4 | 延迟止盈，+30% 强制卖 1/4 |
| 第二批 | 从最高点回撤 5% 卖 1/3 | 同左 |
| 第三批 | 跌破 MA20 或回撤 8% 清仓 | 同左 |

### 4.3 时间止损

买入后 5 个交易日内振幅 < 4% → 主动卖出。

### 4.4 大盘止损

- 大盘跌破 MA20 → 减仓到 50%（每只持仓卖一半，每只只减一次）
- 大盘连续 3 日收盘在 MA60 下方 → 清仓所有股票
- 一个月内因大盘止损被迫减仓 2 次以上 → 空仓观望，等大盘站上 MA60

## 六、风控红线（铁律）

1. **不逆势**：大盘全绿时不建仓
2. **不重仓**：单票 ≤ 20%
3. **不侥幸**：浮亏 > 7% 必须清仓
4. **不频繁**：连续 2 笔止损停止 3 天
5. **不接刀**：连续下跌后不抄底
6. **不追高**：连续拉升第 3 板不追

### 强制冷却（触发任一 → 停止交易 3 天）

- 单日账户总资产回撤 > 3%
- 连续两笔交易触发止损
- 出现情绪化冲动
- 未按系统规则交易

### 紧急情况

| 情况 | 操作 |
|------|------|
| 账户从高点回撤 8% | 全部清仓，100% 理财，休息 1 个月 |
| 连续 3 笔止损 | 暂停 1 周，只做复盘 |
| 踏空焦虑 | 买入 1 天期逆回购，锁仓 3 天 |

## 七、每日流程

### 盘前（8:25 自动）

1. 检查持仓是否有止损/止盈信号
2. 查核心池异动（跌破 MA20 / 主力大幅流出）
3. 大盘 vs MA20/MA60 → GREEN/YELLOW/RED/CLEAR
4. 格式化盘前摘要 → Discord

### 午休（11:55 自动）

1. 持仓涨跌 + 加仓机会判断
2. 买入次数余量
3. → Discord

### 收盘（15:35 自动）

1. 更新持仓最新价格（MX优先）
2. 重算止损/止盈价
3. 检查是否触发止损/止盈
4. 生成明日计划（写明天的日志 MD）
5. → Discord + Obsidian

### 核心池评分（15:40 自动）

1. 批量拉取技术/基本面/资金/舆情数据（MX优先）
2. 四维评分 → 写入 Obsidian 评分报告

### 周复盘（周日 20:00 自动）

1. 本周每笔交易回顾（买入理由/卖出理由/盈亏）
2. 胜率/盈亏比/总收益率统计
3. 选股质量评估（信号真伪分析）
4. 仓位管理复盘（有无违反规则）
5. 系统优化建议

## 十二、常见问题

**Q：妙想 API Key 在哪配置？**
A：项目根目录 `.env` 文件，`MX_APIKEY=your_key`。

**Q：妙想 API 调用失败？**
A：自动 fallback 到 akshare/新浪，日志显示 `[mx_*] 失败`。

**Q：评分数据过期？**
A：非交易时间网络波动导致，全失败返回 `stale: true`。

**Q：临时禁止某只股票？**
A：`config/stocks.yaml` → `blacklist.temporary: [002487]`

**Q：舆情评分怎么算？**
A：MX 搜索研报+评级，基础分 1.5，研报多/买入评级多加分，减持评级扣分。
