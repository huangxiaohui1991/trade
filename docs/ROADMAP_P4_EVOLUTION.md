# 交易系统进化路线图（P4+）

> 起始日期：2026-04-11
> 前置：P1-P3 全部收口，自动化闭环已打通
> 定位：从"能跑起来"进化到"跑得好、跑得稳、跑得聪明"

---

## 当前系统画像

一句话：**决策全自动 + 模拟盘验证 + 实盘手动执行**。

已经有的：
- 四维评分 + 一票否决 + 大盘择时 → 综合决策
- 影子交易自动验证策略信号
- 完整回测框架（sweep / walk-forward / strategy-replay）
- 结构化账本 + 告警中心 + 复盘归因
- 9 个定时 pipeline + Discord 推送 + Hermes CLI 闭环

还没验证的：
- 实盘还没真正跑过完整的买卖周期
- 评分权重和风控参数还是初始值，没经过实盘校准
- 模拟盘刚跑了 1 天（04-10），样本量为零

---

## 进化方向总览

| 阶段 | 主题 | 核心目标 | 预计周期 |
|------|------|---------|---------|
| P4 | 实盘磨合 | 跑通第一轮完整买卖周期，积累真实数据 | 2-4 周 |
| P5 | 参数校准 | 用实盘 + 模拟盘数据校准策略参数 | 持续 |
| P6 | 智能进化 | 自适应调参 + 多策略 + 更深的归因 | 长期 |

---

## P4：实盘磨合期（2-4 周）

> 目标：让系统在真实市场环境中跑起来，暴露问题，积累数据。

### P4-1 交易日历感知

**问题**：当前 cron 不区分节假日，节假日会空跑并产生无效数据。

**方案**：
- [x] 新增 `utils/trading_calendar.py`，维护 A 股交易日历（节假日 + 调休）
- [x] `hermes_cron.sh` 启动时先检查是否交易日，非交易日直接跳过
- [x] pipeline 运行状态记录 `skipped_reason: non_trading_day`
- [x] 数据源：akshare `tool_trade_date_hist_sina()` + 本地缓存（7天刷新）

### P4-2 模拟盘 vs 实盘对账

**问题**：模拟盘和实盘是两套独立的仓位，没有对比机制。

**方案**：
- [x] 月报增加"模拟盘 vs 实盘"对比区块（买入信号数 / 执行数 / 偏离 / 盈亏差异）
- [ ] 收盘报告增加"模拟盘 vs 实盘"对比区块（待实盘有数据后启用）
- [ ] 周报增加模拟盘独立 P&L 统计（待积累更多数据）
- [ ] 当模拟盘信号与实盘操作不一致时，记录 `rule_deviation: shadow_divergence`

### P4-3 月度复盘自动生成

**问题**：`03-复盘/月/2026-04.md` 目前是空模板，需要手动填写。

**方案**：
- [x] 新增 `pipeline/monthly_review.py`
- [x] 从结构化账本自动统计：月度 P&L、胜率、盈亏比、最大回撤
- [x] 从周报聚合：周度汇总表
- [x] 从交易事件提取：亏损最大 3 笔 + 原因分析
- [x] 从 pool action history 提取：核心池变化汇总
- [x] 模拟盘 vs 实盘对比
- [x] 系统参数检查表
- [x] crontab：每月 28 日 20:30 自动执行
- [x] CLI：`bin/trade run monthly --json` / `--month 2026-04`

### P4-4 数据源稳定性加固

**问题**：04-10 实测发现东财数据不稳定，导致评分偏低（6.8/6.9 < 阈值 7）。

**方案**：
- [x] 评分引擎增加 `data_quality` 标记：`ok` / `degraded` / `error`
- [x] 基本面评分 detail 中标注缺失字段（`⚠️缺失:营收,现金流`）
- [ ] `doctor` 增加数据源健康度检查：最近 N 次调用的成功率
- [ ] Discord 推送增加数据质量标记：评分旁边标注 `⚠️ 数据降级` 提示
- [ ] 评分引擎遇到 `data_quality: degraded` 时，用最近一次有效评分作为参考

### P4-5 Hermes 交互增强

**问题**：Hermes-Agent 目前只能通过 CLI 被动调用，缺少主动交互能力。

**方案**：
- [ ] 新增 `trade digest` 命令：一句话总结今日状态（给 Hermes 生成摘要用）
- [ ] `trade suggest` 命令：基于当前状态输出下一步建议（买什么 / 卖什么 / 等待）
- [ ] `trade explain <code>` 命令：解释某只股票的评分明细和决策逻辑
- [ ] 这三个命令的输出格式对 Hermes 友好（结构化 JSON + 人类可读摘要）

## P5：参数校准期（持续）

> 目标：用真实交易数据反哺策略参数，形成"实盘 → 回测 → 调参 → 实盘"闭环。

### P5-1 回测参数自动校准

**前置**：需要至少 20-30 笔闭合交易（约 2-3 个月实盘数据）。

**方案**：
- [ ] 新增 `pipeline/param_calibration.py`
- [ ] 从实盘闭合交易提取真实 MFE/MAE 分布
- [ ] 基于 MFE 分布校准止盈参数（t1_pct / t1_drawdown / t2_drawdown）
- [ ] 基于 MAE 分布校准止损参数（stop_loss / absolute_stop）
- [ ] 基于持仓天数分布校准时间止损（time_stop_days）
- [ ] 输出校准建议报告（不自动修改 strategy.yaml，需人工确认）
- [ ] CLI：`bin/trade calibrate --json`

### P5-2 评分权重优化

**方案**：
- [ ] 收集每笔交易的入场评分 vs 最终盈亏
- [ ] 分析四个维度（技术/基本面/资金/舆情）哪个对盈亏的预测力最强
- [ ] 用 walk-forward 的 sweep 结果交叉验证
- [ ] 输出权重调整建议

### P5-3 选股条件优化

**方案**：
- [ ] 统计核心池股票的"入池后 N 日表现"
- [ ] 分析哪些 veto 规则最有效（拦截了多少真正的亏损）
- [ ] 分析哪些 veto 规则过于严格（误杀了多少盈利机会）
- [ ] 基于数据调整 `screening.mx_query` 和 `scoring.veto` 列表

## P6：智能进化（长期）

> 目标：让系统具备自适应能力，减少人工干预。

### P6-1 自适应风控

**方案**：
- [ ] 根据近期市场波动率动态调整止损幅度（高波动放宽、低波动收紧）
- [ ] 根据账户净值曲线动态调整仓位上限（回撤期自动降仓）
- [ ] 根据连续盈亏状态动态调整买入阈值（连亏后提高门槛）

### P6-2 多策略框架

**方案**：
- [ ] 支持多个独立策略并行运行（不同的评分权重 / 选股条件 / 风控参数）
- [ ] 每个策略独立统计 P&L、胜率、盈亏比
- [ ] 策略间资金隔离，互不影响
- [ ] 定期对比策略表现，淘汰弱策略

### P6-3 深度归因分析

**方案**：
- [ ] 按行业 / 市值 / 持仓天数 / 入场信号类型分组统计盈亏
- [ ] 识别系统的"能力圈"：哪类股票赚钱、哪类亏钱
- [ ] 时间维度分析：哪个时间段（周几 / 月初月末 / 财报季）表现好
- [ ] 输出"策略体检报告"

### P6-4 Web 仪表盘

**方案**：
- [ ] 轻量 Web 服务（FastAPI / Streamlit），读取结构化账本
- [ ] 手机可访问：持仓概览 / 今日决策 / 告警列表 / 模拟盘对比
- [ ] 历史净值曲线 / 回撤图 / 胜率趋势
- [ ] 不做交易操作，只做展示

### P6-5 券商 API 接入（可选）

**方案**：
- [ ] 评估券商 API 可行性（华泰 / 东财 / 同花顺）
- [ ] 先做"半自动"：系统生成订单 → 推送到券商 APP 的条件单
- [ ] 再做"全自动"：系统直接下单（需要严格的风控保护）
- [ ] 全自动模式必须有"熔断开关"：单日亏损超限自动停止

## 优先级建议

### 下周立即做（P4-1 ~ P4-3）

1. **交易日历** — 避免节假日空跑，这是最基础的
2. **月度复盘** — 04 月底就要用，趁现在数据少先把框架搭好
3. **数据源加固** — 04-10 已经暴露了东财数据不稳定的问题

### 本月内做（P4-4 ~ P4-5）

4. **模拟盘对账** — 等模拟盘跑满一周再做，需要数据积累
5. **Hermes 交互增强** — digest / suggest / explain 三个命令

### 持续做（P5）

6. **参数校准** — 需要 20+ 笔闭合交易，预计 2-3 个月后启动

### 看情况做（P6）

7. 自适应风控、多策略、Web 仪表盘 — 等系统稳定运行 3 个月后再考虑

---

## 成功标准

| 阶段 | 标准 |
|------|------|
| P4 完成 | 系统连续运行 4 周无重大故障，模拟盘有 10+ 笔闭合交易 |
| P5 启动 | 实盘有 20+ 笔闭合交易，参数校准报告可输出 |
| P6 启动 | 系统稳定运行 3 个月，月度复盘数据完整 |
