# 选股与评分逻辑说明

> 版本：v2.2（2026-04-11 C版参数更新）
> 数据源：妙想 API 优先 → akshare fallback → 新浪 fallback

---

## 一、整体流程

```
1. 妙想智能选股 API 全市场初筛
   ↓ 返回候选列表（含最新价/MA20/ROE/涨跌幅/量比/成交额/换手率/流通市值等）
2. 第一轮评分：技术面 + 基本面 + 资金流（跳过舆情）
   ↓ 补调 MX 日线算真实均线排列
   ↓ 补调 MX 查营收增长率和现金流
   ↓ 业绩暴雷检测（净利润同比下滑>30% → 一票否决）
3. 按三维分数排序
4. 第二轮：只对前 8 名补充舆情搜索，重算总分
5. 按总分排序 → 分池
   ↓ ≥7 分 → 核心池 | 5-7 分 → 观察池 | <5 分 → 淘汰
6. 写入 config/stocks.yaml + Obsidian
```

入口命令：
```bash
python -m scripts.pipeline.stock_screener --pool all --universe market
```

## 二、第一步：妙想智能选股初筛

当前筛选条件（`config/strategy.yaml` → `screening.mx_query`）：
```
站上20日均线，ROE大于8%，成交额超5000万，股价5-200元，市盈率TTM低于100，剔除ST
```

妙想返回的关键字段：

| 字段 | 用途 |
|------|------|
| 最新价、20日均线 | 技术面：均线趋势判断 |
| 涨跌幅 | 技术面：当日动量（辅助，真实动量用5日） |
| 量比 | 技术面：成交量评分（按市值分档阈值） |
| 换手率 | 资金流：替代成交额绝对值 |
| ROE | 基本面：直接评分 |
| 流通市值 | 量比阈值分档依据 |

---

## 三、第二步：四维评分

满分 10 分，四个维度独立计算后加权汇总。

### 3.1 技术面（满分 3 分，权重 3）

| 子项 | 满分 | 评分规则 |
|------|------|---------|
| 均线趋势 | 0.5 | 站上 MA20 且 MA60 → 0.5；仅站上 MA20 → 0.3；跌破 → 0 |
| 量比（按市值分档） | 0.5 | ≥阈值 → 0.5；≥1.0 → 0.3；<1.0 → 0.1 |
| 动量（5日涨幅） | 0.5 | ≥5% → 0.5；≥2% → 0.3；≥0% → 0.1；<0% → 0 |
| 均线排列（真实计算） | 0.5 | MA5>MA20>MA60 → 0.5；MA20>MA60 → 0.3；否则 → 0 |

量比阈值按市值分档：
- 流通市值 > 200亿：阈值 1.2（大盘股放量标准低）
- 50-200亿：阈值 1.5
- < 50亿：阈值 1.8（小盘股需要更明显放量）

均线排列通过补调 MX 日线（120天收盘价+成交量）计算 MA5/MA20/MA60，不再给固定中性分。

### 3.2 基本面（满分 2 分，权重 2）

| 子项 | 满分 | 评分规则 |
|------|------|---------|
| ROE | 1.0 | ≥15% → 1.0；≥10% → 0.7；≥5% → 0.4；<5% → 0 |
| 营收增长（补调） | 1.0 | ≥20% → 1.0；≥10% → 0.7；≥0% → 0.3；<0% → 0 |
| 现金流（补调） | 1.0 | 经营现金流为正 → 0.5；为负 → 0 |

ROE 从妙想选股结果直接提取。营收增长率和经营现金流通过补调 MX `mx_data` 查询（`"{股票名} 最近一期营收同比增长率 经营活动现金流净额"`），不再给中性分 0.5。补调失败时才 fallback 到 0.5。

### 3.3 资金流（满分 2 分，权重 2）

| 子项 | 满分 | 评分规则 |
|------|------|---------|
| 换手率适配 | 1.0 | 3%-10% → 1.0（健康区间）；>10% → 0.7（异常放量警惕）；1%-3% → 0.4；<1% → 0.2 |
| 北向资金 | 1.0 | 快速模式无数据 → 0.5；传统模式有数据 → 0/0.5/1.0 |

用换手率替代成交额绝对值，避免大盘股和小盘股的绝对值不可比问题。

### 3.4 舆情（满分 3 分，权重 3）

通过 MX 资讯搜索获取研报和机构评级。**只对三维排名前 8 名执行**，其余给默认 1.5 分。

| 指标 | 加分规则 |
|------|---------|
| 基础分 | 1.5 |
| 研报数量 ≥5 | +0.5 |
| 研报数量 ≥2 | +0.3 |
| 买入评级 ≥2 | +0.5 |
| 买入评级 ≥1 | +0.3 |
| 减持评级 ≥2 | -0.5 |

## 四、一票否决

任一触发 → 总分直接归 0：

| 信号 | 触发条件 |
|------|---------|
| `below_ma20` | 最新价 < 20日均线（用真实 MA20） |
| `limit_up_today` | 今日涨幅 ≥ 9.9% |
| `consecutive_outflow` | 连续 3 日主力大额流出 → 硬否决（0 分） |
| `consecutive_outflow_warn` | 连续 3 日流出但站稳 MA20 且成交额 > 5 亿 → 疑似洗盘，减 2 分不否决 |
| `red_market` | 大盘信号 = RED |
| `earnings_bomb` | 净利润同比下滑 > 30%（MX 查询） |
| `ma20_trend_down` | MA20 斜率连续 10 日向下 |

---

## 五、加权总分

```
总分 = 技术面 × (3/3) + 基本面 × (2/2) + 资金流 × (2/2) + 舆情 × (3/3)
     = 技术面 + 基本面 + 资金流 + 舆情
     满分 = 3 + 2 + 2 + 3 = 10
```

---

## 六、分池规则

| 总分 | 分类 | 去向 |
|------|------|------|
| ≥ 6.5 分 | ✅ 可买入 | 核心池（≤3 只） |
| 5-6.5 分 | 🟡 观察 | 观察池（≤5 只） |
| < 5 分 | ❌ 淘汰 | 不入池 |

若候选股全部 < 7 分，核心池留空，不强行买入低分股。

## 八、两种评分模式

### 快速模式（全市场扫描）

两轮评分：
1. 第一轮：妙想选股数据 + 补调日线/财务 → 技术+基本面+资金（跳过舆情）
2. 按三维排序，只对前 8 名补充舆情搜索
3. 重算总分，最终排序

12 只候选约 90 秒完成（含补调日线和舆情搜索）。

### 传统模式（核心池每日评分）

逐个调 `get_technical()` → `get_financial()` → `get_fund_flow()` → `_score_sentiment()`。

适用场景：`core_pool_scoring.py`（每日 15:40，只评 2-3 只核心池股票）。
