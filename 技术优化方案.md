### 1.1 数据层（P0 🟡）

#### P0-1：评分数据源断联 — 核心池全军覆没

**现象：** 核心池3只股票评分全是0（杰瑞/欧陆通）和4（大金重工），全部不通过。

**根因分析：**

```
akshare_data.py 的主力资金 get_fund_flow()
    ↓ 失败（akshare 接口不稳定或网络问题）
    ↓ 返回 {}
stock_screener_pipeline.py 的 score_stock()
    ↓ 读取不到主力数据，flow_score = 0
    ↓ 同时 ROE/营收列名不匹配（硬编码 "2026.04.08"）
    ↓ 基本面分也拿不到
    ↓ 最终输出：总分 = 0~4，全不通过
```

**具体断点：**

| 步骤 | 脚本 | 问题 |
|------|------|------|
| 数据获取 | `akshare_data.py: get_fund_flow()` | akshare 主力资金接口（东财/同花顺）不稳定，经常返回空 |
| 日期硬编码 | `stock_screener_pipeline.py:140` | `"2026.04.08"` 写死，应为动态 `datetime.now().strftime("%Y.%m.%d")` |
| 列名匹配 | `scorer.py` | `roe_col`/`rev_col` 匹配逻辑依赖 mx-screener 输出格式，格式一变就失效 |
| 静默失败 | 全局 `try/except: pass` | 任何一步失败都被吞掉，用户完全不知道数据没拉到 |

**修复方案：** 见 § 二 - 数据层重建

## 二、优化方案（按优先级）

### 2.1 P0 修复：数据层重建

#### 模块：`data_engine.py`（新建）

**目标：** 把 `akshare_data.py` 重构成一个可靠的数据引擎，核心原则：**任何数据都要有 fallback，任何失败都要有日志**。

**架构：**

```
data_engine.py
├── get_realtime_price(codes: list)     # 实时价格（腾讯行情，秒级）
├── get_technical(code)                  # 技术指标（均线/KDJ/MACD）
├── get_fund_flow(code, days)           # 主力资金（多源 fallback）
├── get_financial(code)                  # 基本面（ROE/营收/现金流）
├── get_market_index()                  # 大盘指数（上证/创业板）
└── batch_get(codes)                     # 批量获取，返回 DataFrame
```

**关键设计：**

```python
def get_fund_flow(code: str, days: int = 5) -> dict:
    """主力资金多源 fallback"""

    # Source 1: 东财个股资金流（akshare）
    try:
        df = ak.stock_individual_fund_flow(stock=code, market="sh" if code.startswith("6") else "sz")
        if df is not None and len(df) > 0:
            return parse_eastmoney_flow(df)
    except Exception as e:
        logger.warning(f"东财资金流失败 {code}: {e}")

    # Source 2: 同花顺主力资金
    try:
        df = ak.stock_market_lrb(indicator="净资产收益率")
        # ... 同上
    except Exception as e:
        logger.warning(f"THs资金流失败 {code}: {e}")

    # Source 3: 腾讯行情（备用）
    try:
        return get_tx_fund_flow(code)
    except Exception as e:
        logger.warning(f"腾讯资金流失败 {code}: {e}")

    # Source 4: 返回空但记录
    logger.error(f"全部资金流接口失败 {code}")
    return {"error": "all_sources_failed", "code": code}
```

**关键原则：**
- 每个数据源单独 try/except，不相互影响
- 所有失败写入日志文件（不是 stdout）
- 提供 `data_quality_report()` 方法：哪些股票哪些指标获取失败了，下次跑之前先告知用户
- 加入数据新鲜度检查：如果某数据超过 N 分钟，返回时带上 `stale: True` 标记

@dataclass
class TradeRecord:
    date: str          # 交易日期
    stock_name: str    # 股票名称
    stock_code: str    # 股票代码
    action: str        # BUY / SELL

    hold_days: int     # 持有天数
    exit_reason: str   # 卖出原因：止盈1 / 止盈2 / 止损 / 时间止损 / 清仓
    created_at: str    # 记录时间

def save_trade(record: TradeRecord):
    """追加到 data/交易记录.md"""

def calc_stats(records: list) -> dict:
    """计算胜率、盈亏比、最大单笔亏损等"""
    wins = [r for r in records if r.pnl > 0]
    losses = [r for r in records if r.pnl <= 0]
    return {
        "total_trades": len(records),
        "win_rate": len(wins) / len(records) if records else 0,
        "avg_win": sum(r.pnl for r in wins) / len(wins) if wins else 0,
        "avg_loss": sum(r.pnl for r in losses) / len(losses) if losses else 0,
        "profit_ratio": abs(sum(r.pnl for r in wins) / sum(r.pnl for r in losses)) if losses else 0,
        "max_loss": min(r.pnl for r in records) if records else 0,
        "total_pnl": sum(r.pnl for r in records),
    }
```

# scripts/cron_watchdog.py（新建）
def check_cron_health():
    """每日检查 cron.log，报告是否有错误"""
    log = Path(LOG)
    if not log.exists():
        return "⚠️ cron.log 不存在，cron 可能未运行"
    
    today = datetime.now().date()
    errors = []
    for line in log.read_text().split('\n'):
        if str(today) in line and 'error' in line.lower():
            errors.append(line)
    
    if errors:
        return f"⚠️ 今日 cron 有 {len(errors)} 个错误:\n" + '\n'.join(errors[-5:])
    return "✅ cron 今日运行正常"
```
