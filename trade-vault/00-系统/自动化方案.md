## 二、Cron 任务

由 `scripts/hermes_cron.sh` 统一调度，自动加载 `.env`（MX_APIKEY 等），默认 vault 路径会动态跟随当前仓库根目录；如需覆盖，可显式设置 `AStockVault`。

| 时间 | 命令 | 做什么 | 输出 |
|------|------|--------|------|
| 08:25 周一~五 | `morning` | 大盘数据 + 持仓状态 + 核心池异动 + 模拟盘止损止盈预览 | Discord |
| 09:15 周一~五 | `order overdue-check` | 检查超时未确认条件单（T+1 提醒 / T+2 异常标记） | Discord |
| 09:00-15:30 每30分钟 | `sentiment` | 核心池+持仓舆情扫描 → 关键词告警 | Discord |
| 11:55 周一~五 | `noon` | 持仓涨跌 + 加仓机会 + **模拟盘盘中止损止盈执行** | Discord |
| 15:35 周一~五 | `evening` | 更新持仓价格 + 止损止盈 + 明日计划 + **模拟盘收盘检查** | Discord + Obsidian |
| 15:40 周一~五 | `scoring` | 核心池四维评分 + 写回核心池.md | Obsidian |
| 16:30 周一~五 | `hk_monitor` | 港股遗留仓位价格更新 + 止损检查 | Discord |
| 周日 20:00 | `weekly` | 周报（P&L + 胜率 + 盈亏比） | Obsidian + Discord |
| 每月 28 日 20:30 | `monthly` | 月度复盘（P&L + 模拟盘 vs 实盘 + 参数检查） | Obsidian |

安装 cron：
```bash
crontab config/hermes_crontab
```

## 三、Pipeline 模块

| 模块 | 文件 | 功能 |
|------|------|------|
| 盘前 | `pipeline/morning.py` | 大盘 vs MA20/MA60 → 信号判断，持仓状态，核心池异动，模拟盘预览 |
| 午休 | `pipeline/noon.py` | 持仓涨跌，加仓机会，**模拟盘盘中止损止盈** |
| 收盘 | `pipeline/evening.py` | 更新价格，重算止损止盈，生成明日计划，**模拟盘收盘检查** |
| 评分 | `pipeline/core_pool_scoring.py` | 核心池四维评分（MX 数据 + 搜索） |
| 选股 | `pipeline/stock_screener.py` | tracked / market 双模式选股 → 四维评分 → 严格分池建议 → 同步自选股 → **模拟盘买入** |
| 周报 | `pipeline/weekly_review.py` | 统计本周交易数据，生成周复盘 |
| 影子交易 | `pipeline/shadow_trade.py` | 模拟盘自动买卖，验证交易系统逻辑 |
| 舆情监控 | `pipeline/sentiment_monitor.py` | 核心池+持仓舆情扫描 → 关键词告警 → Discord 推送 |
| 港股监控 | `pipeline/hk_monitor.py` | 港股遗留仓位价格更新 + 止损检查 → Discord 告警 |

补充说明：
- `stock_screener.py --universe tracked`：只在核心池/观察池中重跑
- `stock_screener.py --universe market`：先走全市场扫描，再做四维评分
- 全市场扫描支持 `mx -> akshare -> 本地缓存 -> tracked fallback`

统一入口：
```bash
bin/trade doctor --json
bin/trade status today --json
bin/trade orchestrate morning_brief --json
bin/trade orchestrate close_review --json
bin/trade run morning --json
bin/trade run screener --universe market --pool all --json
```

`trade status today --json` 现在还会返回：
- `today_decision`
- `pool_management.summary`

推荐给 Hermes / OpenClaw 的共享入口：
- `orchestrate morning_brief`
- `orchestrate noon_check`
- `orchestrate close_review`
- `orchestrate weekly_review`
- `orchestrate tracked_scan`
- `orchestrate market_scan`

## 四、影子交易（模拟盘验证）

用东方财富妙想模拟组合自动执行交易信号，验证策略有效性。

| 时机 | 动作 | 说明 |
|------|------|------|
| 选股完成 | 核心池新股 → 模拟盘市价买入 | 每只约 2 万，次日开盘成交 |
| 盘前 8:25 | 预览止损价/止盈价 | 只看不下单（盘前无法交易） |
| 午休 11:55 | 盘中检查 → 触发则执行 | 止损用市价，止盈用限价 |
| 收盘 15:35 | 最后一次检查 → 执行 | 对账 |

手动操作：
```bash
python -m scripts.pipeline.shadow_trade status   # 查看模拟盘
python -m scripts.pipeline.shadow_trade buy       # 买入核心池
python -m scripts.pipeline.shadow_trade check     # 检查止损止盈
python -m scripts.pipeline.shadow_trade report    # 生成报告
```

## 五、数据源优先级

所有数据获取模块统一遵循：**妙想 API 优先 → akshare fallback → 本地缓存回退 → 新浪/历史 fallback**

| 模块 | 优先 | Fallback |
|------|------|---------|
| 选股初筛（market） | 妙想 mx_xuangu | akshare 全市场轻筛 → `data/cache` → tracked fallback |
| 选股初筛（tracked） | tracked 池 + 妙想命中筛选 | tracked 池直接回退 |
| 历史日线 | 妙想 mx_data | akshare → 新浪 |
| 基本面 | 妙想 mx_data | akshare → 新浪 |
| 资金流向 | 妙想 mx_data | akshare → 成交量估算 |
| 舆情评分 | 妙想 mx_search | 默认 1.5 分 |
| 大盘择时 | 妙想 mx_data | akshare |
| 自选股同步 | 妙想 mx_zixuan | — |
| 模拟交易 | 妙想 mx_moni | — |

缓存说明：
- 候选池缓存写入 `data/cache/screening_candidates/`
- 运行状态写入 `data/runtime/daily_state_YYYY-MM-DD.json`
- 池子连续评分状态写入 `data/runtime/pool_state.json`
- 单次运行结果写入 `data/runs/YYYY-MM-DD/`
- 运行锁写入 `data/locks/`
- `data/cache/`、`data/runtime/`、`data/runs/`、`data/locks/` 均不纳入 Git

环境变量配置在项目根目录 `.env`：
```
MX_APIKEY=your_key_here
```

## 七、当前已知边界

- Discord 未配置 webhook 时，pipeline 会继续执行，但状态会记为 `warning`
- 全市场扫描虽然已有缓存回退，但评分阶段仍依赖外部行情/财务接口稳定性
- `data_quality=degraded/error` 已作为新增买入门禁；仍需补最近一次有效评分参考展示
- 港股价格获取依赖 MX/akshare 港股接口稳定性，失败时跳过价格更新
- 目前自动化重点是“辅助决策 + 影子验证”，不是直接实盘下单
- `openclaw` 应只调用 CLI，不直接 import `scripts.pipeline.*`
