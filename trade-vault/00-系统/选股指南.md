> 版本：v1.0

## 一、选股流水线概览

```
选股流程：妙想 API（优先） → AKShare Fallback → 四维评分 → 一票否决 → 分池
```

| 阶段 | 数据源 | 说明 |
|------|--------|------|
| 候选生成 | 妙想 API (`mx.xuangu.search`) | 自然语言条件选股，失败则 AKShare 全市场快照 |
| 候选限定 | `config/stocks.yaml` | 限定核心池 / 观察池 / 全市场 |
| 四维评分 | `scripts/engine/scorer.py` | 技术(3) + 基本面(2) + 资金(2) + 舆情(3) |
| 一票否决 | `scripts/engine/scorer.py` | 触发 veto 信号直接排除 |
| 分池建议 | `scripts/utils/pool_manager.py` | 晋级/保留/降级/移出 |

---

## 二、运行命令

```bash
# 核心池评分（已有持仓股票评分）
bin/trade run scoring

# 选股流水线
bin/trade run screener --pool all --universe tracked   # 已跟踪池（稳定）
bin/trade run screener --pool all --universe market      # 全市场（依赖妙想 API）

# 查看今日决策
cat trade-vault/04-决策/今日决策.md
```

## 三、妙想智能选股

### 配置（`config/strategy.yaml`）

```yaml
screening:
  mx_query: "站上20日均线，ROE大于8%，成交额超5000万，股价5-200元，市盈率TTM低于100，剔除ST"
  mx_select_type: "A股"
  market_scan_limit: 30
  candidate_cache_ttl_hours: 24
```

### 工作流程

1. 用 `mx_query` 调用 `mx.xuangu.search`
2. 妙想返回结果匹配已跟踪池（核心池 + 观察池）
3. 妙想有结果但均不在已跟踪池 → 直接使用妙想结果（全市场选股）
4. 妙想调用失败 → AKShare 全市场快照轻筛

### Fallback 条件（`strategy.yaml`）

```yaml
fallback_filters:
  min_price: 5
  max_price: 200
  min_amount: 50_000_000   # 5000万
  max_pe_ttm: 100           # PE(TTM) > 100 剔除
  exclude_st: true
  ma20_trend_up: true       # MA20 向下直接剔除
```

## 四、四维评分体系

满分 10 分，各维度独立计算后加权求和。

| 维度 | 权重 | 数据来源 | 说明 |
|------|------|----------|------|
| 技术面 | 3 | AKShare | 金叉、量比、RSI、MA 斜率等 |
| 基本面 | 2 | AKShare/妙想 | ROE、营收增速、现金流 |
| 资金流 | 2 | AKShare | 主力净流入、超大单净流入 |
| 舆情 | 3 | 妙想/网络 | 正面新闻、技术形态认可度 |

### 评分阈值

```yaml
scoring:
  thresholds:
    buy: 6.5    # 可买入
    watch: 5    # 观察
    reject: 4   # 拒绝
```

## 五、一票否决（Veto）

触发以下任一信号，直接排除，不参与分池。

| veto key | 中文含义 | 说明 |
|----------|----------|------|
| `below_ma20` | 股价低于 MA20 | 趋势向下 |
| `limit_up_today` | 今日涨停 | 追高风险 |
| `consecutive_outflow` | 连续资金净流出 | 主力撤退 |
| `red_market` | 大盘系统性下跌 | 环境恶劣 |
| `earnings_bomb` | 业绩暴雷 | 基本面风险 |
| `ma20_trend_down` | MA20 趋势向下 | 长期趋势破坏 |

中文标签由 `scripts/engine/scorer.py` 的 `VETO_LABELS` 映射定义。

## 六、股票池管理

### 池子结构（`config/stocks.yaml`）

```yaml
core_pool:    # 核心池：重点关注，分数 >= 6.5
watch_pool:   # 观察池：持续跟踪，分数 >= 5
blacklist:
  permanent: []  # 永久排除
  temporary: []  # 临时排除（可手动添加）
```

### 分池规则

| 操作 | 条件 | 触发 |
|------|------|------|
| 晋级核心池 | 连续 2 天分数 >= 6.5 | 自动晋级 |
| 加入观察池 | 连续 1 天分数 >= 5 | 自动入池 |
| 降级核心池 | 连续 2 天分数 < 5 | 自动降级 |
| 移出观察池 | 连续 2 天分数 < 4 | 自动移出 |
| 一票否决触发 | 任意 veto 信号 | 立即降级 |

### 自动分池运行

```bash
# 收盘后 15:40 核心池自动评分 + 分池
bin/trade run scoring
```
