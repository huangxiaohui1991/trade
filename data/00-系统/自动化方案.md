# A股交易系统自动化方案

## 一、整体架构

```
┌─────────────────────────────────────────────────┐
│                  触发层                           │
│  Kiro Hook (userTriggered)  →  后续: crontab     │
└──────────────┬──────────────────────┬────────────┘
               │                      │
       ┌───────▼───────┐      ┌───────▼───────┐
       │  盘前流程 8:30  │      │  收盘流程 15:30 │
       │ morning_routine│      │ evening_routine│
       └───────┬───────┘      └───────┬───────┘
               │                      │
       ┌───────▼──────────────────────▼───────┐
       │              数据层                    │
       │  AKShare API  ←→  Obsidian MD 文件     │
       │  (行情/基本面)    (持仓/日志/复盘)      │
       └───────┬──────────────────────┬───────┘
               │                      │
       ┌───────▼───────┐      ┌───────▼───────┐
       │   计算引擎      │      │   输出层       │
       │ calculator.py  │      │ 盘前报告 MD    │
       │ scorer.py      │      │ 收盘报告 MD    │
       │ parser.py      │      │ 钉钉/微信推送  │
       └───────────────┘      └───────────────┘
```

## 二、自动化分级

### 第一级：完全自动化（无需人工参与）

| # | 功能 | 输入 | 输出 | 触发时机 | 优先级 |
|---|------|------|------|----------|--------|
| 1 | 大盘状态检查 | AKShare 指数数据 | 上证/创业板 vs 20/60日线，是否可买入 | 盘前 8:30 | P0 |
| 2 | 持仓止损预警 | portfolio.md + AKShare 最新价 | 哪些持仓接近止损价，是否需要挂止损单 | 盘前 8:30 | P0 |
| 3 | 止盈价更新 | portfolio.md + AKShare 最高价 | 移动止盈价重算（从最高点回撤5%/8%） | 收盘后 15:30 | P0 |
| 4 | 时间止损检查 | 持仓买入日期 + 5日K线 | 是否满5天且振幅<3%，提醒卖出 | 收盘后 15:30 | P1 |
| 5 | 核心池技术面监控 | 核心池.md + AKShare | 是否跌破20日线、成交量变化、主力资金异动 | 收盘后 15:30 | P1 |
| 6 | 明日日志+计划生成 | 模板 + 大盘数据 + 持仓数据 + 核心池数据 | 创建明日日志MD，自动填充明日计划（条件单清单、风控状态、核心池检查） | 收盘后 15:30 | P1 |
| 7 | 风控状态检查 | 本周交易记录 + portfolio.md | 买入次数余量、仓位占比、是否冷却中 | 每次买入前 | P0 |
| 8 | 周复盘数据统计 | 本周日志文件 | 胜率、盈亏比、交易次数、最大亏损 | 每周日 | P2 |

### 第二级：半自动化（自动跑数据，人工做决策）

| # | 功能 | 自动部分 | 人工部分 | 触发时机 |
|---|------|----------|----------|----------|
| 1 | 选股打分 | AKShare获取技术面+资金面（7项自动评分） | 确认主线板块、催化、ROE、营收（4项） | 每周日 / 事件驱动 |
| 2 | 核心池调整 | 检测跌破20日线/主力流出，自动提醒 | 决定是否剔除、谁补位 | 收盘后检测到异常时 |
| 3 | 买入决策 | 检查全部买入条件，输出"可买/不可买"及原因 | 最终确认下单 | 盘前 / 午休 |
| 4 | 加仓决策 | 算涨幅、风险检查、优先级排序 | 确认是否加仓 | 午休检查 |
| 5 | 事件驱动更新 | 无法自动获取新闻 | 用户告知消息，系统跑数据评估 | 随时 |

### 第三级：暂不自动化

| 功能 | 原因 |
|------|------|
| 实际下单 | 涉及资金安全，必须人工操作券商APP |
| 新闻监控 | 需要NLP+舆情分析，复杂度高，ROI低 |
| i问财自动筛选 | 无公开API，需要爬虫，维护成本高 |

## 三、核心脚本设计

### daily_routine.py

```python
"""
每日自动化流程
用法:
  python daily_routine.py morning    盘前流程（8:30）
  python daily_routine.py noon       午休检查（12:00）
  python daily_routine.py evening    收盘流程（15:30）
  python daily_routine.py weekly     周复盘（周日）
"""

def morning_routine():
    """盘前流程 8:30
    注意：今日日志已在昨日收盘时生成（含明日计划），盘前只补充实时数据
    """
    # 1. 获取实时大盘数据（开盘前竞价阶段）
    market = check_market_status()
    
    # 2. 读取今日日志中的明日计划（昨晚生成的）
    today_journal = load_today_journal()
    
    # 3. 补充实时数据到今日日志（更新大盘最新状态）
    update_journal_morning(today_journal, market)
    
    # 4. 检查是否有隔夜异常（如美股大跌、重大新闻）
    # → 需人工确认，系统只提醒检查
    
    # 5. 输出盘前摘要（直接从明日计划提取，不单独生成报告）
    summary = extract_morning_summary(today_journal)
    
    # push_to_dingtalk(summary)  # 后续: 手机推送
    
    return summary


def noon_check():
    """午休检查 12:00"""
    # 1. 持仓是否达到加仓条件
    add_opportunities = check_add_conditions()
    
    # 2. 本周剩余买入次数
    remaining = get_weekly_buy_remaining()
    
    # 3. 条件单成交情况（需人工确认）
    return {
        "add_opportunities": add_opportunities,
        "remaining_buys": remaining,
    }


def evening_routine():
    """收盘流程 15:30"""
    # 1. 更新持仓最新价格
    update_portfolio_prices()
    
    # 2. 重算止损止盈价
    recalc_all_stops()
    
    # 3. 时间止损检查
    time_stop_alerts = check_time_stops()
    
    # 4. 核心池异常检测（跌破20日线等）
    watchlist_alerts = check_watchlist_alerts()
    
    # 5. 更新今日日志收盘部分
    update_journal_evening()
    
    # 6. 生成明日计划（写入明天的日志文件）
    tomorrow_plan = generate_tomorrow_plan(
        market=get_market_status(),
        portfolio=load_portfolio(),
        watchlist=load_watchlist(),
        risk=check_risk_status(),
        time_stops=time_stop_alerts,
    )
    create_tomorrow_journal(tomorrow_plan)
    
    # 7. 生成收盘报告
    report = generate_evening_report(
        time_stop_alerts, watchlist_alerts, tomorrow_plan
    )
    
    save_report(report)
    return report


def weekly_review():
    """周复盘（周日）"""
    # 1. 统计本周数据
    stats = calc_weekly_stats()
    
    # 2. 生成周复盘MD
    generate_weekly_review(stats)
    
    # 3. 核心池/观察池技术面刷新
    refresh_watchlist_scores()
    
    return stats
```

  "version": "1.0.0",
  "description": "每日盘前自动运行交易系统检查，生成盘前报告",

## 五、后续演进路线

### Phase 1: Kiro Hook 手动触发（当前目标）
- 创建 daily_routine.py
- 配置 3 个 userTriggered Hook（盘前/收盘/周复盘）
- 你在 Kiro 里点一下按钮就跑
- 预计工作量：1-2 天

### Phase 2: macOS 定时任务
- 用 crontab 或 launchd 定时执行
- 盘前 8:30 / 收盘 15:30 / 周日 20:00
- 报告输出到 Obsidian 文件，打开就能看
- 预计工作量：半天

### Phase 3: 手机推送
- 接入钉钉机器人 Webhook 或企业微信机器人
- 盘前报告 + 止损预警 + 收盘摘要推送到手机
- 上班时不用开电脑也能收到关键信息
- 预计工作量：半天

### Phase 4: 智能化增强（远期）
- 接入新闻 API 做舆情监控（核心池相关新闻自动推送）
- 基于历史数据回测系统参数
- 自动生成条件单参数（导出为券商可导入的格式）
- 预计工作量：1-2 周
