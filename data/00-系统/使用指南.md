# A股交易助手 使用指南

## 快速开始

### 环境准备

```bash
# 安装 AKShare（已安装可跳过）
pip install akshare

# 验证安装
python -c "import akshare; print(akshare.__version__)"
```

### Obsidian 设置

1. 用 Obsidian 打开 `.kiro/skills/a-stock-trading/data/` 作为 Vault
2. 安装推荐插件：
   - **Dataview** — 自动汇总数据（仪表盘依赖此插件）
   - **Templater** — 模板自动填充日期
   - **Calendar** — 日历视图快速跳转日志
3. 设置 Templater 模板目录为 `00-系统/模板/`

## 每日执行流程（上班族版）

> 核心原则：系统自动准备数据，你只需确认执行。每天实际操作时间 ≤ 15 分钟。

### 详细步骤

#### 盘前确认（8:30-9:15）— 👤 人工 5 分钟

| 步骤 | 操作 | 自动/人工 | 说明 |
|------|------|----------|------|
| 1 | 打开 Obsidian 今日日志 | 👤 | 文件已由昨晚收盘自动创建 |
| 2 | 看底部"明日计划"的大盘状态 | 👀 只看 | 🤖 昨晚自动填好 |
| 3 | 看"明日条件单清单" | 👀 只看 | 🤖 昨晚自动算好止损/止盈价 |
| 4 | 在券商 APP 挂条件单 | 👤 手动 | 照着清单挂，不需要自己算价格 |
| 5 | 看隔夜新闻标题（5分钟） | 👤 手动 | 只看核心池+观察池相关 |
| 6 | 朗读心理检查表 | 👤 手动 | |
| 7 | 打勾确认 | 👤 打勾 | 在日志盘前部分打 ✅ |

**你不需要做的：** 查大盘数据、算止损价、算止盈价、判断风控状态 — 全部已自动完成。

#### 午休检查（12:00-12:30）— 👤 人工 2 分钟

| 步骤 | 操作 | 自动/人工 | 说明 |
|------|------|----------|------|
| 1 | 看条件单是否成交 | 👤 看券商APP | 1 分钟 |
| 2 | 如有加仓机会，挂限价单 | 👤 手动 | 明日计划里有加仓条件和剩余次数 |

**你不需要做的：** 算加仓条件、查买入次数 — 明日计划里都有。

#### 收盘记录（15:00-15:30）— 👤 人工 3 分钟 + 🤖 自动 5 分钟

| 步骤 | 操作 | 自动/人工 | 说明 |
|------|------|----------|------|
| 1 | 填账户总资产 | 👤 手动 | 从券商 APP 看，填到日志 |
| 2 | 填今日交易记录 | 👤 手动 | 如果有成交的话 |
| 3 | 写"今日一句话" | 👤 手动 | 情绪记录 |
| 4 | 对 Kiro 说"收盘记录" | 👤 触发 | 一句话触发以下全部自动流程 |
| 5 | 获取大盘收盘数据 | 🤖 自动 | AKShare market_status |
| 6 | 更新持仓最新价格 | 🤖 自动 | AKShare technical |
| 7 | 重算止损/止盈价 | 🤖 自动 | calculator.py |
| 8 | 检查时间止损 | 🤖 自动 | 5日振幅 < 3% 提醒 |
| 9 | 检查核心池异常 | 🤖 自动 | 跌破20日线提醒 |
| 10 | 生成明日计划 | 🤖 自动 | 写入明天的日志文件 |

**你不需要做的：** 查行情、算价格、检查风控、创建明天的日志 — 全部自动。

#### 特殊情况

| 情况 | 处理 |
|------|------|
| 全天无法操作 | 前一天晚上条件单已挂好，自动执行。收盘后补做"收盘记录" |
| 条件单未成交 | 收盘后检查，系统会在明日计划中重新列出 |
| 触发加仓但没时间 | 错过就错过，不追高。系统追求长期期望值 |
| 重大新闻/事件 | 午休时对 Kiro 说"帮我查下 XXX"，快速评估 |

### 一、选股打分（核心功能）

#### 自动打分（推荐）

对 Kiro 说：
> "帮我给 601127 打分"
> "给贵州茅台打个分"

Kiro 会运行：
```bash
python scripts/akshare_data.py score 601127
```

输出包含：
- **自动评分项**（7项）：技术面（站上20日线、60日线方向）、成交量（得分+突破量）、资金面（北向、主力）、现金流
- **需人工确认项**（4项）：是否主线板块、是否有催化、ROE是否达标、营收是否增长
- **原始数据**：完整的技术面/基本面/资金面数据供参考

示例输出：
```
自动得分 6.5/10.5，还有 4 项需人工确认。技术面通过✅

自动评分明细：
✅ 股价站上20日均线 (+1)
✅ 60日均线向上 (+1)
✅ 成交量条件 (+1.0)
✅ 北向资金近5日净流入 (+1)
✅ 无连续主力大额卖出 (+1)
✅ 经营现金流为正 (+1)
⚠️ 属于当前主线板块 (待确认)
⚠️ 有政策或产业催化 (待确认)
⚠️ ROE≥8% (待确认)
⚠️ 营收连续两季增长 (待确认)
```

然后你补充人工确认项，Kiro 会给出最终得分和建议。

#### 批量打分

对 Kiro 说：
> "帮我批量打分：600519,000858,601127,002594,300750"

```bash
python scripts/akshare_data.py batch_score 600519,000858,601127,002594,300750
```

输出按得分排序，快速筛选出技术面最好的标的。

#### 手动打分

如果你已经有所有数据，可以直接用打分引擎：
```bash
python scripts/scorer.py '{"stock_name":"贵州茅台","stock_code":"600519","is_main_sector":true,"has_catalyst":false,"roe_pass":true,"revenue_growth":true,"cash_flow_positive":true,"above_ma20":true,"ma60_up_or_flat":true,"volume_score":1.0,"northbound_inflow":true,"no_major_outflow":true,"volume_breakout":false}'
```

### 二、大盘状态检查

对 Kiro 说：
> "看下大盘状态"
> "现在能买吗"

```bash
python scripts/akshare_data.py market_status
```

输出：
```json
{
  "上证指数": {
    "close": 3919.28,
    "MA20": 3998.77,
    "MA60": 4070.84,
    "above_MA20": false,
    "below_MA60_days": 13
  },
  "_summary": {
    "status": "清仓信号"  // 可买入 / 谨慎（减半买入）/ 清仓信号
  }
}
```

**状态含义：**
- `可买入`：上证或创业板站上20日线，正常买入
- `谨慎（减半买入）`：均未站上20日线，但未触发清仓，买入金额减半
- `清仓信号`：连续3日在60日线下方，应清仓

### 三、仓位计算

#### 首次买入

对 Kiro 说：
> "我想买贵州茅台，现在 1800 块，帮我算下仓位"

```bash
python scripts/calculator.py position 1800
```

输出：
```
建议买入: ¥28,800（16股）
止损价: ¥1,710.00（-5%成本止损）
绝对止损: ¥1,674.00（-7%无条件止损）
第一批止盈: ¥1,944.00（+8%）
第一次加仓触发: ¥1,890.00（+5%）
第二次加仓触发: ¥1,980.00（+10%）
最大亏损: ¥2,016（占总资金0.25%）✅
```

#### 带20日线的精确计算

```bash
# position <买入价> <总资金> <当前股票总市值> <该股当前市值> <20日线价格> <是否例外> <本周已买次数>
python scripts/calculator.py position 1800 800000 30000 0 1780 false 0
```

#### 加仓判断

对 Kiro 说：
> "贵州茅台可以加仓吗？现在涨到 1900 了"

```bash
# add <首次买入价> <当前均价> <持有股数> <当前价> <加仓价> [更多参数]
python scripts/calculator.py add 1800 1800 16 1900 1890 800000 28800 28800 1870 true 1 0 1920
```

输出会告诉你：
- 是否满足加仓条件（涨幅≥5%）
- 加仓后新均价和新止损价
- 加仓后如果止损，总亏损是否超过总资金2%
- 加仓优先级得分（多只股票时用于比较）

### 五、每日工作流

> 详细流程见上方"每日执行流程"章节，以下为快速参考。

#### 盘前 → 对 Kiro 说"盘前检查"
系统读取昨晚生成的明日计划，补充实时大盘数据，输出盘前摘要。

#### 午休 → 对 Kiro 说"午休检查"
检查持仓加仓条件和本周剩余买入次数。

#### 收盘 → 对 Kiro 说"收盘记录"
一句话触发全部收盘自动流程：更新价格 → 重算止损 → 检查异常 → 生成明日计划。

### 六、每周复盘

每周日晚上，对 Kiro 说 **"周复盘"**

Kiro 会：
1. 读取本周所有日志（`parser.py journal 02-日志/ 7`）
2. 统计胜率、盈亏比、交易次数
3. 基于模板生成 `03-复盘/周/2026-W15.md`
4. 提醒你更新核心池

然后对 Kiro 说 **"帮我筛选下周核心池"**
- 批量打分候选股
- 按得分排序推荐

## 数据说明

### AKShare 接口稳定性

| 接口 | 交易时间 | 非交易时间 | 说明 |
|------|---------|-----------|------|
| stock_zh_a_hist（日线） | ✅ 稳定 | ✅ 稳定 | 技术面数据主要来源 |
| stock_zh_index_daily（指数） | ✅ 稳定 | ✅ 稳定 | 大盘状态主要来源 |
| stock_zh_a_spot_em（实时） | ✅ 稳定 | ⚠️ 可能超时 | 有 fallback 到日线 |
| stock_individual_info_em | ✅ 稳定 | ✅ 稳定 | 个股基本信息 |
| stock_financial_analysis_indicator | ✅ 稳定 | ✅ 稳定 | 财务指标 |
| stock_hsgt_north_net_flow_in_em | ✅ 稳定 | ⚠️ 偶尔超时 | 北向资金 |
| stock_individual_fund_flow | ✅ 稳定 | ⚠️ 偶尔超时 | 个股资金流向 |

**注意：** 东方财富接口在非交易时间（晚上/周末）可能有限流，部分请求会超时。这不影响使用，重试即可。

### 数据更新频率

- **日线数据**：每日收盘后更新（15:00 后）
- **实时行情**：交易时间内实时
- **财务数据**：季报发布后更新（通常滞后 1-2 个月）
- **北向资金**：每日收盘后更新
- **资金流向**：每日收盘后更新

## 常见问题

**Q: AKShare 报错 ConnectionError？**
A: 东方财富接口限流，等几秒重试。非交易时间更容易出现。

**Q: 基本面数据不完整？**
A: 部分小盘股或次新股财务数据可能不全，需要手动补充。

**Q: 如何修改总资金？**
